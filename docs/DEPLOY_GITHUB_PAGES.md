# Деплой на GitHub Pages

## Инструкция по загрузке проекта

### Вариант 1: Через Git (рекомендуется)

1. **Откройте терминал/командную строку** в папке проекта `E:\Nord`

2. **Инициализируйте Git репозиторий:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

3. **Подключите GitHub репозиторий:**
   ```bash
   git branch -M main
   git remote add origin https://github.com/ВАШ_USERNAME/ВАШ_РЕПОЗИТОРИЙ.git
   git push -u origin main
   ```

4. **Настройте GitHub Pages:**
   - Откройте репозиторий на GitHub
   - Перейдите в **Settings** → **Pages**
   - В разделе **Source** выберите `main` (или `master`) ветку
   - Нажмите **Save**

5. **Подождите 1-2 минуты** и ваш сайт будет доступен по адресу:
   `https://ВАШ_USERNAME.github.io/ВАШ_РЕПОЗИТОРИЙ/`

### Вариант 2: Через GitHub Desktop

1. Скачайте [GitHub Desktop](https://desktop.github.com/)
2. Откройте GitHub Desktop
3. **File** → **Add Local Repository**
4. Выберите папку `E:\Nord`
5. Если появится ошибка "not a git repository", нажмите **create a repository**
6. Сделайте commit всех файлов
7. **Publish repository** в верхнем меню
8. Следуйте шагу 4 из Варианта 1

### Вариант 3: Через веб-интерфейс GitHub (если не получается первые два)

1. Создайте репозиторий на GitHub
2. **НЕ ЗАГРУЖАЙТЕ папку Nord целиком!**
3. Загружайте **СОДЕРЖИМОЕ папки Nord** (все файлы и папки ВНУТРИ Nord):
   - Перетащите все файлы и папки из `E:\Nord\` в окно GitHub
   - Или используйте кнопку "Upload files" и выберите всё содержимое
4. Важно: в корне репозитория должен быть `index.html`, а не папка `Nord/index.html`

## Важные моменты

✅ **Правильная структура в репозитории:**
```
репозиторий/
  ├── index.html
  ├── catalog.html
  ├── css/
  ├── js/
  ├── images/
  └── ...
```

❌ **Неправильная структура:**
```
репозиторий/
  └── Nord/
      ├── index.html
      └── ...
```

## После загрузки

1. Обновите файл `robots.txt` - замените `https://example.com` на реальный адрес вашего сайта
2. Проверьте, что сайт работает
3. Если возникают проблемы с путями, возможно нужно будет добавить базовый путь в HTML

## Проверка

Ваш сайт будет доступен по адресу:
- `https://ВАШ_USERNAME.github.io/НАЗВАНИЕ_РЕПОЗИТОРИЯ/`

Если репозиторий называется `ВАШ_USERNAME.github.io`, то сайт будет по адресу:
- `https://ВАШ_USERNAME.github.io/`

## Файлы для GitHub Pages

- `.gitignore` - список файлов, которые не нужно загружать
- `.nojekyll` - отключает обработку Jekyll на GitHub Pages
