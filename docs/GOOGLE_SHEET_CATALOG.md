# Google Таблица: структура каталога

Есть два варианта. Если у вас уже сделана таблица “по‑русски” (как в примере), используйте **Вариант A**.

## Вариант A (как в примере): один лист `Каталог` с русскими колонками

Лист: `Каталог`

Заголовки в 1-й строке (точно как названия колонок):
- `ID`
- `Категория`
- `Название`
- `Краткое описание`
- `Цена аренды`
- `Цена покупки`
- `Изображение`
- `Характеристики`
- `В наличии`
- `На главной`

Правила:
- **`Категория`**: например `Экскаваторы`, `Бульдозеры`, `ИТП` (можно расширять).
- **`Характеристики`**: одна ячейка строкой, например: `Вес: 21 т, Мощность: 125 л.с., Глубина: 6.5 м`.
  - Apps Script разобьет строку на список по разделителям `,` или `|` или `;`.
- **`В наличии` / `На главной`**: `Да/Нет` (также понимает `TRUE/FALSE`).
- **`Изображение`**: имя файла, например `excavator1.jpg`. Файл положить в `images/equipment/`.
  - Если нужно несколько фото: перечислите через `|`, например `excavator1.jpg | excavator1-2.jpg | excavator1-3.jpg`.

## Вариант B: 2 листа `categories` + `items` (англ. заголовки)

Этот вариант удобен, если хотите хранить стабильные `id` категорий.

### 1) Лист `categories`

Колонки:
- `id` — идентификатор (латиница, без пробелов), например `excavators`
- `name` — отображаемое имя, например `Экскаваторы`

### 2) Лист `items`

Колонки (минимум):
- `id` (число)
- `category` (строка — должна совпадать с `categories.id`)
- `name`
- `shortDesc`
- `fullDesc`
- `priceRent`
- `priceBuy`
- `image` (имя файла, например `excavator-1.jpg`; файл кладем в `images/equipment/`)
- `specs` (строка со списком, например: `Вес: 21 т | Мощность: 125 л.с. | Глубина: 6.5 м`)
- `available` (`TRUE/FALSE`)
- `featured` (`TRUE/FALSE`)
- `showOnMain` (`TRUE/FALSE`)

Примечания:
- `specs` в Apps Script будет разбиваться по разделителю `|` в массив.
- Если `image` пустая — карточка покажет заглушку.
